Demonstrates validation error messages containing tokens using the new replaceToken code.

Now each validation method can have a companion method that is passed the error message string. It replaces tokens within that string and returns the updated message.

Tokens can be textual and therefore meaningful (like {VALUE}) as compared to the existing model ({0}).

The default token replacement method supports the numeric tokens of the past, and adds these: {LABEL} and {VALUE}. {LABEL} extracts the label associated with the element, either from the data-msglabel attribute on the element or from a <label for= > tag. {VALUE} is the actual data entered.

Validators with minimum and maximum rules have additional tokens: {MIN} and {MAX}. The length validators also have {LENGTH} and {DIFF}. {DIFF} is the number the user has exceeded the min or max.
